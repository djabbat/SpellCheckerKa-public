# SpellCheckerKa — Концепция экосистемы проверки орфографии грузинского языка

## Мультиплатформенное решение: Web + Mobile + Browser Extension

---

## 1. Введение

### 1.1. Название
**SpellCheckerKa** (SpellChecker + Ka — код Грузии)

### 1.2. Краткое описание
SpellCheckerKa — это экосистема инструментов для проверки орфографии грузинского языка, охватывающая все основные платформы использования текста: веб-интерфейс для работы с документами, нативные мобильные приложения для письма на ходу и браузерные расширения для проверки текста в любом месте интернета.

### 1.3. Проблема
- Отсутствие качественных инструментов проверки грузинской орфографии на всех платформах
- Пользователи пишут тексты везде: в браузере, в мессенджерах, в мобильных приложениях, в документах
- Существующие решения (Microsoft Word, Grammarly) имеют слабую поддержку грузинского языка
- Нет единой экосистемы, где пользователь может проверять тексты в любом контексте

### 1.4. Решение
SpellCheckerKa предлагает три взаимодополняющих продукта:

| Продукт | Назначение | Ключевая ценность |
|---------|------------|-------------------|
| **Web-приложение** | Работа с документами, импорт/экспорт .docx | Полнофункциональный редактор с аналитикой |
| **Мобильное приложение** | Проверка текста на iOS/Android | Письмо в мессенджерах, заметках, SMS |
| **Browser Extension** | Проверка текста в любом поле ввода браузера | Универсальность: соцсети, email, формы |

| .docx | fflate + DOMParser | Полноценный импорт/экспорт Word-файлов |
| Словарь | ETS + user_words.txt | 993 589 слов, пользовательские дополнения |
| Аутентификация | Phoenix Auth + OAuth | Email, Google, Facebook |

**Ключевые функции:**
- Редактор с подсветкой ошибок в реальном времени

| Endpoint | Method | Описание | Rate Limit |
|----------|--------|----------|------------|
| `POST /api/check` | POST | Проверка текста | 30/мин (anon), 60/мин (auth) |
| `POST /api/check/batch` | POST | Пакетная проверка (до 100 текстов) | 10/мин (auth) |
| `POST /api/dictionary/add` | POST | Добавление слова в пользовательский словарь | 50/мин |
| `POST /api/dictionary/remove` | POST | Удаление слова из словаря | 50/мин |
| `GET /api/dictionary/list` | GET | Получение пользовательского словаря | 30/мин |
| `POST /api/sync` | POST | Синхронизация настроек между устройствами | 20/мин |
| `GET /api/stats` | GET | Статистика использования | 10/мин |
| `POST /api/auth/register` | POST | Регистрация пользователя | 5/мин |
| `POST /api/auth/login` | POST | Вход в систему | 10/мин |

### 5.4. Офлайн-режим (Mobile)

| Компонент | Размер | Обновление |
|-----------|--------|------------|
| Сжатый словарь (основной) | ~15 MB | Еженедельно при WiFi |
| Пользовательский словарь | ~50 KB | Синхронизация при подключении |

**Стратегия синхронизации:**
1. При отсутствии сети — проверка по локальному словарю

3. Конфликты разрешаются в пользу облачной версии

## 6. Технические риски и их mitigation

| Риск | Вероятность | Влияние | Mitigation |
|------|-------------|---------|------------|
| **Производительность ETS** при 100k+ одновременных пользователей | Средняя | Высокое | Шардирование по tenant, Redis кэш для горячих данных, горизонтальное масштабирование |
| **Совместимость расширения** с разными версиями браузеров | Высокая | Среднее | Автоматическое тестирование на BrowserStack (Chrome, Firefox, Safari, Edge), feature detection, graceful degradation |
| **React Native производительность** на старых устройствах | Средняя | Среднее | Оптимизация рендеринга (useMemo, useCallback), FlatList вместо ScrollView, минимальные анимации, тестирование на iPhone 8 / Android 8 |
| **Apple App Store Review** для keyboard extension | Низкая | Высокое | Тщательное следование Human Interface Guidelines, демонстрация уникальной ценности, подготовка видео-демо для ревьюеров |
| **OCR точность** для грузинского текста | Средняя | Среднее | Tesseract.js с кастомным train data, пост-обработка результатов, fallback на ручной ввод |
| **Конфликты синхронизации** между устройствами | Средняя | Среднее | CRDT подход, временные метки, разрешение в пользу последнего изменения, UI для разрешения конфликтов |
| **DDoS / злоупотребление API** | Низкая | Высокое | Rate limiting, API keys, Cloudflare WAF, мониторинг аномалий |
| **Зависимость от сторонних библиотек** | Средняя | Среднее | Регулярный аудит безопасности (npm audit, mix deps.audit), минимальные зависимости, форки критических библиотек |
| **Производительность mirror overlay** при больших текстах (10k+ слов) | Средняя | Среднее | Виртуализация, рендеринг только видимой области, Web Workers для анализа |
